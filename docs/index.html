Features

Everything you need to
work remotely

From stateful shell sessions to full PTY interactive mode — JARVIS has you covered wherever you are.

WhatsApp & Telegram

Send shell commands directly from a WhatsApp or Telegram chat. The bot responds with formatted output, exit codes, working directory, and elapsed time.

Interactive PTY Mode

Prefix any command with ! to start a real PTY session. Run !gemini, !node, or any REPL — the process sees isTTY=true and enters interactive mode.

Web Dashboard

React + xterm.js dashboard at localhost:3000. Manage operators, scan WhatsApp QR, view live terminal output, browse command history.

Flutter Mobile App

Native iOS & Android app with cyberpunk dark/light UI. WebSocket connection with exponential backoff, interactive PTY banner with Ctrl+C/D/Enter, and local command history.

Secure by Default

JWT-protected dashboard, per-operator whitelist, configurable rate limiting, command timeout kill, and execution mode control (ai / restricted / full-shell).

Persistent Shell Sessions

Each operator gets their own PowerShell session. cd changes stick across commands — your working directory is preserved between messages.

How it works

Up and running in minutes

No cloud accounts, no subscriptions. Install on your machine, scan a QR, and your terminal becomes reachable from any phone.

1

Install & Configure

Clone the repo, run npm install, create your .env file with your JWT secret and dashboard password. Requires Node.js 22+.

2

Connect Your Bots

Open the dashboard, scan the WhatsApp QR code with your phone, and optionally add your Telegram bot token. Add your phone number to the operator whitelist.

3

Control from Anywhere

Send any shell command from WhatsApp or Telegram. Use ! prefix for interactive sessions. Watch live output in the dashboard terminal.

Message Flow

User (WhatsApp/Telegram)
   │ "git status"
   ▼
[IMessengerAdapter] ──► RouteMessageUseCase
                               │
                  AuthorizeOperator ──► No → "Not authorized"
                               │
                  ShellSession.execute("git status")
                               │
                  PowerShell wrapper → output + exitCode + cwd
                               │
                  Reply to chat + WebSocket broadcast → Dashboard
Installation

Get started in 5 minutes

JARVIS runs entirely on your machine. No cloud, no external dependencies beyond Node.js.

1

Clone & Install

Requires Node.js 22+, PowerShell (Windows 10/11 included)

2

Configure Environment

Create .env with your JWT secret, dashboard password, and optional Telegram token

3

Build Dashboard

Compile the React dashboard into static files served by Fastify

4

Start & Connect

Run npm run dev, open the dashboard, scan the WhatsApp QR code

bash
# Clone and install $ git clone https://github.com/jarvis-terminal/jarvis.git $ cd jarvis $ npm install $ cd dashboard && npm install && cd ..
.env
# Dashboard DASHBOARD_PORT=3000 DASHBOARD_PASSWORD=your_secure_password JWT_SECRET=min_32_chars_random_secret_here # Database DB_PATH=./data/jarvis.db # WhatsApp WA_AUTH_PATH=./data/whatsapp-auth # Telegram (optional) TELEGRAM_BOT_TOKEN=123456:ABC... # Terminal DEFAULT_CWD=C:\Users\YOU INITIAL_MODE=full-shell RATE_LIMIT_PER_MINUTE=10
bash
# Build dashboard, then start $ cd dashboard && npm run build && cd .. $ npm run dev # Expected output: JARVIS Dashboard: http://localhost:3000 INFO Server listening at http://127.0.0.1:3000

Mobile App

Native mobile terminal

A full Flutter app with cyberpunk design, WebSocket connection, and interactive PTY support — without WhatsApp or Telegram.

Direct WebSocket Connection

Connects straight to your JARVIS backend over WebSocket with JWT auth, exponential backoff reconnect (2s→30s), and 30-second keepalive pings.

Interactive PTY Banner

When an interactive process starts, a control bar appears with Ctrl+C, Ctrl+D, Enter, and Esc buttons — sending real PTY signals to the backend.

Quick Commands Bar

9 one-tap chips for common commands: dir, git status, !gemini, !node, and more.

Cloudflare Tunnel Ready

Enter any HTTPS URL — the app auto-converts to wss://. Use a Cloudflare Tunnel for secure remote access without port forwarding.

Yes. WhatsApp and Telegram already route through their servers, so messaging works anywhere. For the web dashboard and Flutter mobile app, use a Cloudflare Tunnel — enter your HTTPS tunnel URL in the app and it auto-converts to wss://. No port forwarding needed.
Set COMMAND_TIMEOUT_SECONDS in your .env (default: 60). When the timeout fires, the shell session kills the process and responds with a timeout error. For interactive PTY sessions, use ⌃C via the Telegram keyboard buttons or Flutter app controls.
